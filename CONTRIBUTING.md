Thank you for your interest in contributing! This project welcomes contributions from the community.

- [Code of Conduct](#code-of-conduct)

## Code of Conduct

Be respectful, inclusive, and constructive in all interactions.

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Make your changes
4. Add tests for new functionality
5. Run the test suite (`pytest tests/`)
6. Commit your changes with clear messages
7. Push to your branch
8. Open a Pull Request

## Development Setup

```bash
# Clone the repo
git clone https://github.com/jkelleman/semantic-metrics-modeling-assistant.git
cd semantic-metrics-modeling-assistant

# Install dependencies
pip install -e .
pip install pytest

# Run tests
pytest tests/
```

## Areas for Contribution

-  **Bug fixes** - Help identify and fix issues
-  **Documentation** - Improve guides and examples
-  **Features** - Implement items from the enhancement docs
-  **Tests** - Increase test coverage
-  **UX improvements** - Better error messages and output formatting

## Pull Request Guidelines

- Keep PRs focused on a single change
- Include tests for new functionality
- Update documentation as needed
- Follow existing code style
- Write clear commit messages

## Questions?

Open an issue for discussion before starting major changes.
